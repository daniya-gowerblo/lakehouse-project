# Лабораторная работа 3: Lakehouse на Polars + Delta Lake

## Описание
Данная работа посвящена построению полноценного пайплайна данных (Data Pipeline) по архитектуре **Lakehouse** для прогнозирования задержек авиарейсов. В качестве инструментов обработки данных используются библиотеки **Polars** (для высокопроизводительных вычислений) и **Delta Lake** (для управления версиями данных и обеспечения ACID-транзакций). Обучение моделей машинного обучения и логирование метрик осуществляется через платформу **MLflow**.

### Цель работы
1. Реализовать многоуровневую архитектуру хранения данных: **Bronze** (сырые данные), **Silver** (очищенные и нормализованные данные), **Gold** (агрегаты и признаки для ML).
2. Использовать возможности Delta Lake: инкрементальная загрузка (`append`), слияние данных (`MERGE`), очистка старых версий (`VACUUM`).
3. Обучить модели регрессии (предсказание минут задержки) и классификации (факт задержки > 15 мин) с логированием результатов в MLflow.

---

## Архитектура пайплайна

### 1. Bronze Layer (Raw Data)
*   **Задача:** Загрузка исходных CSV-файлов без изменений структуры.
*   **Реализация:** Данные считываются из папки `data/raw` и записываются в Delta-таблицу в режиме `append`. Добавляются мета-колонки (`source_file`, `ingestion_timestamp`) для аудита происхождения данных.
*   **Особенности:** Поддержка эволюции схемы (`schema_mode="merge"`), что позволяет добавлять новые колонки в будущих батчах без ошибок.

### 2. Silver Layer (Cleaned & Enriched)
*   **Задача:** Очистка от пропусков, фильтрация отмененных рейсов, создание производных признаков.
*   **Трансформации:**
    *   Фильтрация: `Cancelled == 0`, `ArrDelay IS NOT NULL`.
    *   Признаки: `hour` (из времени вылета), `season` (зима/весна/лето/осень), `route` (Origin-Dest).
    *   Нормализация: Переименование колонок в нижний регистр для единообразия.
*   **Интеграция:** Используется операция **`MERGE`** (Upsert) для обновления существующих записей и добавления новых на основе уникального ключа рейса (`FlightDate`, `Airline`, `FlightNum`, `Origin`, `Dest`). Это предотвращает дублирование данных при повторном запуске пайплайна.
*   **Партиционирование:** Таблица разделена по колонкам `year` и `month` для ускорения чтения временных диапазонов (Partition Pruning).
*   **Обслуживание:** Выполняется команда `VACUUM` для удаления старых версий файлов, не относящихся к текущему состоянию таблицы.

### 3. Gold Layer (Business & ML Aggregates)
Создаются две независимые витрины:
1.  **Analytics (`analytics_agg`):** Агрегированные данные о средних задержках по авиакомпаниям, аэропортам и сезонам. Используется для BI-отчетности.
2.  **ML Features (`ml_features`):** Таблица с числовыми и категориальными признаками, подготовленная для обучения моделей. Целевая переменная для классификации: `is_delayed` (1 если `ArrDelay > 15`, иначе 0).

### 4. ML Pipeline
*   **Модели:**
    *   **Регрессия:** `RandomForestRegressor` для предсказания точного времени задержки в минутах. Метрика: MSE (Mean Squared Error).
    *   **Классификация:** `RandomForestClassifier` для предсказания факта существенной задержки. Метрика: Accuracy.
*   **Логирование:** Все параметры моделей, метрики качества и сами артефакты (сохраненные модели `.pkl`) отправляются в MLflow Tracking Server. Каждый этап обучения фиксируется как отдельный Run внутри эксперимента `Flight_Lab3`.

## Инструкция по запуску

### Предварительные требования
*   Установленный [Docker Desktop](https://www.docker.com/products/docker-desktop/).
*   Python 3.10+ и пакетный менеджер `pip` (или `conda`).

### Шаг 1. Подготовка данных
1. Скачайте датасет [US Flight Delays](https://www.kaggle.com/datasets/usdot/flight-delays) или аналогичный.
2. Поместите CSV-файлы в папку `data/raw/`.
   * *Примечание:* Для тестирования достаточно одного файла небольшого размера (например, за один месяц).

### Шаг 2. Установка зависимостей
Установите необходимые библиотеки локально:
```bash
pip install polars deltalake mlflow scikit-learn pandas pyarrow boto3
```

### Шаг 3. Запуск MLflow Server
В отдельном окне терминала запустите сервер трекинга через Docker:
```bash
docker-compose up mlflow
```
Сервер будет доступен по адресу: `http://localhost:5000`

### Шаг 4. Запуск пайплайна
Во втором окне терминала выполните главный скрипт:
```bash
python src/main.py
# или
py src/main.py
```

### Шаг 5. Просмотр результатов
1. Откройте браузер и перейдите по адресу `http://localhost:5000`.
2. Выберите эксперимент **`Flight_Lab3`**.
3. Изучите метрики (MSE, Accuracy) и скачайте обученные модели из раздела Artifacts.
